OmniVoice — Concurrent Stream Tester

Fires N WebSocket streams (optionally staggered by a fixed delay) all sending the same text. Each stream's first-chunk latency, server gen time, and audio metrics are shown live in the table. Audio plays from the stream you pick. Chunking happens entirely server-side; the server fires all chunks into the batcher at t=0 so later chunks generate in the background while the client plays the earlier ones — no "wait for previous chunk to finish" gaps.
